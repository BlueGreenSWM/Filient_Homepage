# 조건 유형

Filient는 다양한 조건을 조합하여 정교한 파일 관리 규칙을 만들 수 있습니다.

## 파일 속성 조건

### 파일 이름
파일명 기반 필터링:
- **정확히 일치**: 특정 이름과 완전히 일치
- **포함**: 특정 문자열 포함
- **시작**: 특정 문자열로 시작
- **끝**: 특정 문자열로 끝남
- **정규식**: 고급 패턴 매칭

```
예시:
- "IMG_"로 시작하는 파일
- ".tmp"로 끝나는 파일
- "2024"를 포함하는 파일
```

### 파일 확장자
파일 형식별 분류:
- **단일 확장자**: .jpg, .pdf, .docx
- **다중 확장자**: 여러 확장자 동시 선택
- **확장자 그룹**: 이미지, 문서, 동영상 등
- **확장자 없음**: 확장자가 없는 파일

```
지원 그룹:
- 이미지: jpg, png, gif, bmp, svg
- 문서: pdf, doc, docx, txt, md
- 동영상: mp4, avi, mov, mkv
- 오디오: mp3, wav, flac, aac
```

### 파일 크기
크기 기반 필터:
- **정확히**: 특정 크기와 일치
- **이상**: 최소 크기 설정
- **이하**: 최대 크기 설정
- **범위**: 최소-최대 범위 설정

```
단위 옵션:
- Bytes (B)
- Kilobytes (KB)
- Megabytes (MB)
- Gigabytes (GB)
```

## 시간 조건

### 생성 날짜
파일 생성 시점 기준:
- **특정 날짜**: 정확한 날짜 지정
- **이전**: 특정 날짜 이전
- **이후**: 특정 날짜 이후
- **기간**: 날짜 범위 설정
- **상대적**: 최근 X일/주/월

### 수정 날짜
마지막 수정 시점:
- **오늘**: 오늘 수정된 파일
- **어제**: 어제 수정된 파일
- **이번 주**: 이번 주 수정
- **이번 달**: 이번 달 수정
- **사용자 정의**: 직접 기간 설정

### 접근 날짜
마지막 접근 시점:
- **최근 접근**: X일 이내 접근
- **오래된 파일**: X일 이상 접근 안 함
- **기간 설정**: 접근 기간 범위

## 위치 조건

### 폴더 경로
특정 위치 지정:
- **정확한 경로**: 특정 폴더만
- **하위 포함**: 하위 폴더 모두 포함
- **제외 경로**: 특정 경로 제외
- **다중 경로**: 여러 경로 동시 지정

### 폴더 깊이
계층 구조 기준:
- **최상위**: 루트 폴더만
- **깊이 제한**: 최대 X단계까지
- **특정 깊이**: 정확히 X단계
- **최소 깊이**: X단계 이상

## 내용 조건

### 텍스트 포함
파일 내용 검색:
- **키워드**: 특정 단어 포함
- **문구**: 정확한 문구 일치
- **정규식**: 패턴 검색
- **대소문자**: 구분/무시 옵션

### 메타데이터
파일 속성 정보:
- **작성자**: 파일 작성자
- **태그**: 파일 태그
- **설명**: 파일 설명
- **저작권**: 저작권 정보

## 시스템 조건

### 파일 상태
시스템 속성:
- **숨김 파일**: 숨김 속성 파일
- **시스템 파일**: 시스템 파일
- **읽기 전용**: 수정 불가 파일
- **실행 파일**: 실행 가능 파일

### 권한
접근 권한:
- **소유자**: 특정 사용자 소유
- **그룹**: 특정 그룹 소속
- **권한 레벨**: 읽기/쓰기/실행

### 중복 파일
중복 감지:
- **이름 중복**: 같은 이름
- **내용 중복**: 같은 내용 (해시)
- **유사 파일**: 유사도 기준

## 특수 조건

### 파일 형식
특정 형식 감지:
- **텍스트/바이너리**: 파일 유형
- **인코딩**: UTF-8, ASCII 등
- **압축 파일**: ZIP, RAR 등
- **암호화**: 암호화된 파일

### 애플리케이션
연결 프로그램:
- **기본 앱**: 기본 실행 프로그램
- **최근 사용**: 최근 열어본 앱
- **특정 앱**: 지정 앱으로 열기

## 조건 조합

### AND 조건
모든 조건 충족:
```
파일 크기 > 10MB AND
수정 날짜 < 30일 AND
확장자 = PDF
```

### OR 조건
하나 이상 충족:
```
확장자 = JPG OR
확장자 = PNG OR
확장자 = GIF
```

### NOT 조건
조건 제외:
```
폴더 = Downloads AND
NOT 확장자 = TMP
```

### 복합 조건
중첩된 조건:
```
(크기 > 100MB OR 생성일 > 1년) AND
(확장자 = ZIP OR 확장자 = RAR) AND
NOT 폴더 = Archive
```

## 조건 템플릿

### 자주 사용하는 조건
- **오래된 대용량 파일**: 크기 > 100MB AND 수정일 > 6개월
- **임시 파일**: 이름 시작 "~" OR 확장자 = TMP
- **스크린샷**: 이름 포함 "Screenshot" OR "스크린샷"
- **다운로드 파일**: 폴더 = Downloads AND 생성일 < 30일

### 사용자 정의 템플릿
자주 사용하는 조건 저장:
1. 조건 설정
2. 템플릿으로 저장
3. 이름 지정
4. 재사용

## 성능 고려사항

### 효율적인 조건
- 파일명/확장자 조건 우선
- 크기 조건 다음
- 내용 검색 최후

### 피해야 할 조건
- 너무 광범위한 조건
- 중첩이 깊은 복합 조건
- 대용량 파일 내용 검색

## 모범 사례

### 조건 설계 원칙
1. **명확성**: 이해하기 쉬운 조건
2. **효율성**: 빠른 처리 가능
3. **정확성**: 의도한 파일만 선택
4. **유지보수**: 수정이 용이한 구조
