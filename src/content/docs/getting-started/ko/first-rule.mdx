# 첫 번째 규칙 만들기

이 가이드에서는 Filient에서 첫 번째 자동화 규칙을 만드는 과정을 단계별로 안내합니다.
다운로드 폴더의 PDF 파일을 자동으로 문서 폴더로 이동하는 간단한 규칙을 만들어보겠습니다.

## 📁 시작하기 전에: 폴더 추가하기

Filient를 처음 실행하면 관리할 폴더가 없는 빈 상태입니다. 규칙을 만들기 위해서는 먼저 관리할 폴더를 추가해야 합니다.

### 왜 폴더를 추가해야 하나요?

Filient는 선택된 폴더만 모니터링하여 시스템 성능을 최적화합니다. 전체 시스템을 스캔하는 대신, 필요한 폴더만 선택적으로 관리하여 효율적인 파일 정리가 가능합니다.

### 1단계: 폴더 추가 인터페이스 찾기

Filient 메인 화면을 열면 다음과 같은 구성을 볼 수 있습니다:

- **왼쪽 사이드바**: 폴더 목록과 메뉴 영역
- **폴더 영역**: 추가된 폴더 관리 영역

왼쪽 사이드바 상단에서 **"폴더 관리"** 섹션을 찾습니다.

![폴더 추가 버튼 위치](/docs/images/folder-selection.png)

### 2단계: 새 폴더 추가하기

#### 폴더 추가 버튼 클릭
- 폴더 관리 섹션 헤더 오른쪽의 **➕ (플러스)** 버튼을 클릭합니다

#### macOS 폴더 선택 대화상자
- 버튼을 클릭하면 macOS 표준 폴더 선택 창이 나타납니다

#### 폴더 선택 팁
- 🎯 **추천 시작 폴더**: 다운로드, 데스크탑, 문서
- ⚠️ **피해야 할 폴더**: 시스템, 라이브러리, 애플리케이션
- 💡 **스마트 선택**: 가장 어지러운 폴더부터 시작하세요

### 3단계: 폴더 추가 확인

폴더를 선택하고 **"열기"** 버튼을 클릭하면:

1. **사이드바에 폴더 표시**
   - 폴더 아이콘과 이름이 목록에 나타남
   - 하위 폴더가 있는 경우 펼침 화살표 표시

![추가된 폴더 목록](/docs/images/folder-added.png)

## 🎯 첫 번째 규칙 만들기

이제 폴더를 추가했으니, 자동화 규칙을 만들어보겠습니다.

### 1. 규칙 생성 시작하기

#### 폴더 선택
1. 사이드바에서 방금 추가한 폴더를 클릭합니다
2. 폴더를 선택하면 우측에 해당 폴더의 규칙 목록이 표시됩니다
3. 우측 상단의 **"새 규칙 생성하기"** 버튼을 클릭합니다

![규칙 생성 버튼](/docs/images/rule-add-button.png)

#### 규칙 생성 방법 선택

규칙 생성 대화상자가 나타나면 두 가지 방법 중 하나를 선택할 수 있습니다:

- 📋 **템플릿 사용**: 미리 만들어진 템플릿 활용
- 📝 **수동으로 만들기** + **AI 어시스턴트** : 조건과 액션을 직접 설정 (새 규칙 만들기 버튼)

이번 예제에서는  **"새 규칙 만들기"** 를 선택합니다.

![규칙 생성 방법 선택](/docs/images/rule-creation-methods.png)

### 2. 기본 정보 입력

규칙 생성 화면이 나타나면 기본 정보를 입력합니다.

![규칙 기본 정보 입력](/docs/images/rule-info.png)

#### 규칙 이름
명확하고 이해하기 쉬운 이름을 지정합니다:
```
PDF 문서 정리
```

#### 규칙 설명 (선택사항)
나중에 규칙의 목적을 쉽게 파악할 수 있도록 설명을 추가합니다:
```
다운로드 폴더의 PDF 파일을 문서 폴더로 자동 이동
```

#### 규칙 옵션
![규칙 기본 정보 입력](/docs/images/rule-options.png)
- ⬜ **하위 폴더 포함**: 폴더 내 모든 하위 폴더에도 규칙 적용
- ⬜ **실행 전 승인 여부**: 실행 전 사용자 승인을 받을 것인지 체크
- ⬜ **규칙을 템플릿으로 저장하기**: 작성된 규칙을 템플릿에 저장할 것인지 여부 체크 (일회성)

### 3. 조건 설정

조건은 어떤 파일에 규칙을 적용할지 결정합니다.

![조건 설정 화면](/docs/images/condition.png)

#### 조건 추가하기
1. **"➕ 조건 추가"** 버튼을 클릭합니다
2. 드롭다운에서 조건 유형을 선택합니다: **"파일 확장자"**
3. 세부 설정을 입력합니다: `문서`
4. **"확인"** 버튼을 클릭합니다

![조건 설정 화면](/docs/images/condition2.png)

#### 추가 조건 예시 (선택사항)
더 정교한 필터링을 원한다면 조건을 추가할 수 있습니다:

**파일 크기 조건**:
- 조건 유형: **"파일 크기"**
- 연산자: **크거나 같음**
- 값: **100 KB**

이렇게 하면 100KB 이상의 PDF 파일만 이동됩니다.

```
규칙 내의 모든 조건은 AND로 연결되며, 하나의 조건 안에 있는 요소들은 OR 조건으로 묶입니다.
```

### 4. 액션 설정

액션은 조건에 맞는 파일에 어떤 작업을 수행할지 결정합니다.

![액션 설정 화면](/docs/images/action.png)

#### 액션 추가하기
1. **"➕ 액션 추가"** 버튼을 클릭합니다
2. 액션 유형을 선택합니다: **"파일 이동"**
3. 대상 폴더를 설정합니다:
   - **폴더 경로**: `~/Documents`
   - 또는 **"폴더 선택"** 버튼으로 직접 선택
4. **"확인"** 버튼을 클릭합니다

![액션 설정 화면 2](/docs/images/action2.png)

#### 예외 처리
![폴더 없으면 생성](/docs/images/move_option.png)

액션의 우측 톱니바퀴 버튼을 누르면 상세설정을 볼 수 있습니다.

대상 폴더(~/Documents/PDF)가 없는 경우:
- ✅ **자동 생성**: Filient가 자동으로 폴더를 만듭니다
- 계층 구조도 자동 생성 가능 (예: ~/Documents/2024/PDF)

파일이 겹쳐 충돌하는 경우:
   - **건너뛰기**: 동일한 이름의 파일이 있으면 이동하지 않음
   - **덮어쓰기**: 기존 파일을 새 파일로 교체
   - **이름 변경** (권장): 숫자를 붙여 저장 (예: document(1).pdf)

### 5. 저장 및 테스트

#### 규칙 저장
1. 모든 설정을 다시 한 번 확인합니다
2. **"저장"** 버튼을 클릭합니다
3. 규칙이 목록에 추가되고 활성화됩니다

#### 테스트 실행 (권장)

![테스트 버튼](/docs/images/test_button.png)

실제 실행 전에 테스트로 결과를 미리 확인할 수 있습니다:

1. 저장된 규칙 우측의 점 3개를 선택하여 메뉴를 엽니다.
2. **"테스트"** 버튼을 클릭합니다
3. 시뮬레이션 결과를 확인합니다:
   - 영향받을 파일 수
   - 성공 / 실패 여부 등
4. 결과가 만족스러우면 **"실행"** 을 통해 실행하거나 스케줄링을 통해 정기적 실행이 가능합니다.
![테스트 실행 결과](/docs/images/test.png)

### 6. 실행 및 결과 확인

#### 수동 실행
1. 규칙을 선택하고 **"지금 실행"** 버튼을 클릭합니다
2. 실행이 완료되면 실행 요약을 확인할 수 있습니다.
   - 성공/실패/건너뜀 카운트

#### 실행 기록
모든 실행 내역은 히스토리에 저장됩니다:
1. **"히스토리"** 탭을 클릭합니다
2. 실행 기록을 확인할 수 있습니다
3. 각 기록을 클릭하면 상세 정보를 볼 수 있습니다
4. 필요시 **"롤백"** 기능으로 작업을 되돌릴 수 있습니다

---

## 🎉 축하합니다!

첫 번째 자동화 규칙을 성공적으로 만들었습니다! 이제 다운로드 폴더에 PDF 파일이 추가되면 자동으로 Documents/PDF 폴더로 이동됩니다.

### 다음 단계

이제 기본을 마스터했으니 더 고급 기능을 시도해보세요:

#### 📚 더 복잡한 규칙 만들기
- 여러 액션 연결 (이동 → 이름 변경 → 태그)
- 여러 조건 연결

#### ⏰ 자동화 설정
- **스케줄러**: 매일 오전 9시에 자동 실행
- **일괄 처리**: 여러 규칙을 순차적으로 실행

#### 🎨 템플릿 만들기
자주 사용하는 규칙을 템플릿으로 저장:
1. 규칙 선택 → **"템플릿으로 저장"**
2. 템플릿 이름과 설명 입력
3. 다른 폴더에서도 재사용 가능

### 유용한 규칙 예시

#### 📸 스크린샷 정리
- **조건**: 파일명이 "스크린샷" 또는 "Screenshot"으로 시작
- **액션**: ~/Pictures/Screenshots/2024 폴더로 이동

#### 🗑️ 임시 파일 청소
- **조건**: 확장자가 .tmp, .temp, .cache
- **액션**: 30일 이상 된 파일은 휴지통으로

#### 📊 프로젝트 파일 정리
- **조건**: 특정 프로젝트 키워드 포함
- **액션**: 프로젝트 폴더로 이동 + 프로젝트 태그 추가

### 문제 해결

#### ❓ 규칙이 실행되지 않아요
- 규칙이 **활성화**되어 있는지 확인
- 조건이 올바르게 설정되었는지 확인
- 폴더 접근 권한이 있는지 확인
- 로그에서 오류 메시지 확인

#### ❓ 잘못된 파일을 이동했어요
- **히스토리** 탭에서 해당 실행 찾기
- **"롤백"** 버튼으로 원상 복구
- 규칙 조건을 수정하여 재발 방지
